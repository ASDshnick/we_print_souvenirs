# Тест-план ручного тестирования

**Проект:** We Print Souvenirs  
**Роль:** тестировщик  
**Тестируемая ветка:** `testing-branch`  
**Объект тестирования:** backend API на Spring Boot

## 1. Цель тестирования

Проверить работоспособность основных backend-сценариев приложения: регистрацию и авторизацию пользователей, работу личного кабинета, создание и просмотр заказов, административные операции, корзину, чат по заказу и корректность ограничения доступа.

## 2. Область тестирования

В тестирование входят:

- регистрация пользователя;
- вход пользователя и получение JWT;
- просмотр профиля;
- изменение пароля;
- изменение личных данных;
- создание заказа;
- просмотр заказов пользователя;
- просмотр пользователей администратором;
- блокировка и разблокировка пользователя;
- изменение роли пользователя;
- сброс пароля администратором;
- просмотр и обновление заказов администратором;
- добавление товаров в корзину;
- просмотр и удаление товаров из корзины;
- история сообщений чата по заказу;
- проверка доступа к защищённым эндпоинтам.

Не входят в тестирование:

- UI фронтенда;
- нагрузочное тестирование;
- тестирование платёжных систем;
- тестирование внешних облачных хранилищ.

## 3. Тестовое окружение

- Java 21;
- Spring Boot 4;
- PostgreSQL;
- Flyway;
- Gradle;
- Postman, Insomnia или `curl` для ручного API-тестирования.

## 4. Предусловия

- база данных PostgreSQL запущена;
- в корне проекта настроен `.env` с переменными:
  - `DB_NAME`;
  - `DB_USER`;
  - `DB_PASS`;
  - `JWT_SECRET_KEY`;
- приложение успешно запускается;
- миграции Flyway применяются без ошибок;
- создан хотя бы один пользователь с ролью `USER`;
- создан хотя бы один пользователь с ролью `ADMIN`.

## 5. Тестовые сценарии

| ID | Сценарий | Шаги | Ожидаемый результат |
|---|---|---|---|
| TC-01 | Регистрация пользователя | Отправить `POST /user/register` с username, password, email | Ответ `201 Created`, пользователь создан |
| TC-02 | Повторная регистрация | Отправить регистрацию с уже существующим username | Возвращается ошибка, дубль не создаётся |
| TC-03 | Авторизация | Отправить `POST /user/login` с корректными данными | Ответ `200 OK`, в теле есть JWT |
| TC-04 | Профиль пользователя | Отправить `GET /user/profile` с JWT | Возвращаются username, email, phone |
| TC-05 | Смена пароля | Отправить `PUT /user/change-password` со старым и новым паролем | Пароль изменён, ответ `200 OK` |
| TC-06 | Неверный старый пароль | Повторить смену пароля с неверным oldPassword | Ответ `400 Bad Request` |
| TC-07 | Изменение данных | Отправить `PUT /user/change-data` с новыми username/email/phone | Данные пользователя обновлены |
| TC-08 | Создание заказа | Отправить `POST /user/orders` с типом заказа и требованиями | Ответ `201 Created`, заказ создан со статусом `NEW` |
| TC-09 | Создание заказа без типа | Отправить заказ без поля `type` | Ответ `400 Bad Request` |
| TC-10 | История заказов | Отправить `GET /user/orders` с JWT | Возвращается список заказов пользователя |
| TC-11 | Просмотр пользователей админом | Отправить `GET /admin/users` с ADMIN JWT | Возвращается список пользователей |
| TC-12 | Доступ USER к админке | Отправить `GET /admin/users` с USER JWT | Доступ запрещён |
| TC-13 | Блокировка пользователя | Отправить `PUT /admin/users/{id}/block` | Поле `blocked` меняется |
| TC-14 | Изменение роли | Отправить `PUT /admin/users/{id}/role` | Роль пользователя обновляется |
| TC-15 | Сброс пароля | Отправить `POST /admin/users/{id}/reset-password` | Возвращается временный пароль |
| TC-16 | Просмотр заказов админом | Отправить `GET /admin/orders` | Возвращается список заказов |
| TC-17 | Фильтр заказов | Отправить `GET /admin/orders?userId=...&status=...` | Возвращаются только подходящие заказы |
| TC-18 | Обновление заказа | Отправить `PUT /admin/orders/{id}` со статусом и процентом | Заказ обновляется |
| TC-19 | Некорректный процент | Передать `completionPercentage` меньше 0 или больше 100 | Возвращается ошибка |
| TC-20 | Статус READY | Передать статус `READY` | Процент выполнения автоматически становится `100` |
| TC-21 | Добавление в корзину | Отправить `POST /cart/add` | Товар добавлен в корзину |
| TC-22 | Просмотр корзины | Отправить `GET /cart/items` | Возвращаются позиции и общая сумма |
| TC-23 | Удаление из корзины | Отправить `DELETE /cart/items/{itemId}` | Позиция удалена |
| TC-24 | Чат по заказу | Отправить сообщение в чат заказа | Сообщение сохраняется и доступно в истории |
| TC-25 | История чата без доступа | Попытаться получить историю чужого заказа | Доступ запрещён |

## 6. Критерии приёмки

- критичные пользовательские сценарии выполняются без ошибок;
- защищённые эндпоинты недоступны без нужной роли;
- миграции БД выполняются на чистой базе;
- автотесты проходят успешно;
- покрытие тестами превышает 60%.
