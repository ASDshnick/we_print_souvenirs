# Отчёт о ручном тестировании

**Проект:** We Print Souvenirs  
**Ветка:** `testing-branch`  
**Роль:** тестировщик  
**Тип тестирования:** ручное функциональное тестирование backend API

## 1. Краткое резюме

Было проведено ручное тестирование основных backend-сценариев приложения. Проверялись регистрация и авторизация пользователя, работа профиля, создание заказа, просмотр заказов, административные функции, миграции базы данных и базовые ограничения доступа.

По итогам проверки основные критичные сценарии работают корректно. Административная часть, создание заказов и миграции Flyway прошли проверку. Найденные ранее проблемы с маршрутом `GET /user/orders`, лишним `@PathVariable` в смене пароля и запуском Flyway были исправлены.

## 2. Окружение

- Java 21;
- Spring Boot 4;
- Gradle;
- PostgreSQL в Docker;
- Flyway;
- ручные запросы через `curl`.

## 4. Исправленные дефекты

| ID | Дефект | Статус |
|---|---|---|
| BUG-01 | Дублирующий маршрут `GET /user/orders` в двух контроллерах | Исправлен |
| BUG-02 | Лишний `@PathVariable("id")` в `PUT /user/change-password` | Исправлен |
| BUG-03 | Flyway не запускался в Spring Boot 4 без `spring-boot-starter-flyway` | Исправлен |
| BUG-04 | Порт `8080` был занят старым процессом при повторном запуске | Учтено при проверке |

## 5. Замечания и риски

- Без токена некоторые защищённые эндпоинты могут возвращать `403`, а не `401`. Это не блокирует основной функционал, но может быть доработано через `AuthenticationEntryPoint`.
- Часть ручных проверок выполнялась через `curl`; при передаче кириллицы в Windows-терминале возможны проблемы с кодировкой. Через Postman или Insomnia такой проблемы быть не должно.
- Нагрузочное тестирование и тестирование UI не проводились.

## 6. Итог

Критичные backend-сценарии прошли ручную проверку. Приложение готово к дальнейшему тестированию фронтенда и расширению тестового покрытия по новым функциональным требованиям.
